# 모바일 메신저 대화 로그 분석기 개발

## 📋 프로젝트 개요

디지털 포렌식 연구원으로서 범죄 용의자의 스마트폰에서 추출한 메신저 대화 로그 파일(.txt)을 분석하기 위한 Windows 분석 도구(Viewer)를 개발합니다.

## 🎯 개발 목표

수사관이 구조화되지 않은 텍스트 형태의 대화 로그 데이터를 쉽고 빠르게 조회하고 분석할 수 있는 도구 제작

## 📂 샘플 데이터

`Sample_Log.txt` 파일을 생성하여 아래 내용을 저장하세요:

```plaintext
[2024-11-20 14:01:05] 김철수 : 안녕하세요, 이번 프로젝트 관련해서 문의드립니다.
[2024-11-20 14:02:10] 이영희 : 네, 철수님. 어떤 부분이 궁금하신가요?
[2024-11-20 14:05:33] 김철수 : 포렌식 데이터 추출 속도가 조금 느린 것 같아서요.
[2024-11-20 14:06:12] 박민수 : (삭제된 메시지입니다)
[2024-11-20 14:10:55] 이영희 : 그 부분은 최적화 작업을 진행 중입니다. 걱정 마세요.
[2024-11-21 09:00:00] 시스템 : ** 대화방이 생성되었습니다 **
[ERROR] 데이터 손상됨 (복구 불가능)
[2024-11-21 09:15:22] 김철수 : 알겠습니다. 그럼 믿고 맡기겠습니다. 파일 전송합니다.
[2024-11-21 09:15:25] 김철수 : FILE_SEND: secret_plan.pdf
[2024-11-21 09:20:40] 박민수 : 확인했습니다. 보안에 신경 써주세요.
```

> ⚠️ **참고**: 데이터가 오염되어 일부 형식이 맞지 않는 줄이 포함되어 있습니다.

## 📌 개발 요구사항

### 필수 기능 (70점)

#### 1. 파일 로드 (File Load)
- 파일 열기 버튼을 통해 로컬의 텍스트 파일 선택 및 읽기
- `OpenFileDialog` 사용

#### 2. 데이터 파싱 (Data Parsing)
- 각 라인을 `[시간]`, `[발신자]`, `[메시지 내용]`으로 분리
- 형식이 맞지 않는 라인 처리:
  - 예: `[ERROR]...`, 시스템 메시지 등
  - 제외 또는 별도 처리하여 프로그램 안정성 확보
- 예외 처리 구현 필수

#### 3. 리스트 출력 (Display)
- 파싱된 데이터를 `DataGrid` 또는 `ListView`에 출력
- 컬럼 구성:
  - `Date` (날짜/시간)
  - `Sender` (발신자)
  - `Message` (메시지 내용)

### 추가 기능 (30점) ⭐

#### 4. 검색 필터링 (Search)
- 상단에 검색창(`TextBox`) 배치
- 키워드 입력 시 메시지 내용 또는 발신자에 해당 키워드가 포함된 행만 필터링
- **LINQ 활용 권장**

#### 5. 특이사항 강조 (Highlight)
- 메시지 내용 중 "파일 전송" 또는 "FILE" 키워드 포함 시:
  - 글자색을 빨간색으로 표시, 또는
  - 배경색 변경
- **목적**: 수사관이 중요 증거를 빠르게 식별

## 🛠 기술 스택 권장사항

- **언어**: C#
- **프레임워크**: WPF 또는 WinForms
- **UI 컴포넌트**: DataGrid, TextBox, Button, OpenFileDialog
- **데이터 처리**: LINQ, 정규표현식(Regex)
